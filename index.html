Etiquetas

Raíz

1. !DOCTYPE

Indica el tipo de documento

Usamos, por ejemplo, !DOCTYPE html para indicar que se trata de un documento html y por ende saber qué sintaxis y gramática utiliza su código fuente.

2. html

Engloba todo el documento

Usamos esta etiqueta para englobar nuestro codigo html.

3. Comentario

Sólo visible en el código fuente

Usamos esta etiqueta para hacer comentarios en el codigo.

Metadatos

4. head
Delimita el encabezado del documento <head><title>mi head < /title> < /head>
5. meta
metainformación sobre el documento <meta charset="UTF-8"/>
6. title
título del documento <title>mi titulo < /title>
7. link
enlace a otros archivos <link rel="stylesheet" href="style.css"/>
8. style
hoja de estilo incluida en el documento <style>h1{color: purple;}</style>

Secciones

9. body

delimita el cuerpo del documento

<body>mi contenido < /body>
10. section

La etiqueta section está pensada para agrupar los apartados del contenido principal del documento

<section>mi seccion < /section>
11. article

La etiqueta article está pensada para agrupar el contenido de la página que forma una unidad en sí misma desde el punto de vista temático.

<article>< h1>Google</h1> <p>informacion...</p> </article>
12. nav

La etiqueta nav está pensada para agrupar los enlaces a otras páginas o los enlaces a los apartados de la propia página.

<nav> <ul> <li>Inicio</li> </ul> </nav>
13. aside

La etiqueta aside está pensada para agrupar contenido secundario y tangencial al contenido al que acompaña (por ejemplo, un bloque de anuncios, un grupo de enlaces externos relacionados, una cita del texto).

14. h1 - h6

Las etiquetas h1, h2, h3, h4, h5 y h6 están pensadas para marcar los títulos y subtítulos de los apartados de un documento.

<h1>mi encabezado < /h1>
15. header

Representa un grupo de ayudas introductorias o de navegación.

<header> <nav> <ul> <li>Inicio</li> </ul> </nav> < /header>
16. footer

La etiqueta footer está pensada para contener información general sobre el documento, información que se suele poner al final del documento: autor, direcciones de contacto, etc.

<footer> <a href="link" >Twitter< /a> < /footer>
17. address

La etiqueta address está pensada para contener información de contacto del autor de la información (dirección de correo, teléfono, etc.), pero no otro tipo de información.

<address> puedes contactarme al 809-123-4567 < /address>

20. div

Define una división. Esta etiqueta permite agrupar varios elementos de bloque (párrafos, encabezados, listas, tablas, divisiones, etc).

21. blockquote

Está pensada para identificar una cita larga, que puede contener varios párrafos u otras etiquetas.

<blockquote> <p> <Antiguamente eran las universidades independientes entre sí, y hasta del Gobierno mismo: cada cual tenía su régimen, sus estudios, sus métodos y aun sus pretensiones distintas: no sólo disponían arbitrariamente de sus fondos, sino que hasta era también arbitraria en ellas la enseñanza.> </p> </blockquote>
22. pre

Se utiliza cuando se quiere conservar los espacios en blanco, los saltos de línea y los tabuladores del texto original. Las etiwuetas html que he mostrado fueron utilizando esta etiqueta.

23. main

Se utiliza para agrupar el contenido principal de una página, sin incluir los elementos comunes a otras páginas como bloques de navegación, información general del sitio, etc..

24. figure

Muestra una ilustracion.

25. figcaption

representa un subtítulo o leyenda asociado al contenido del elemento padre figure , pudiendo ser colocado como primer o último hijo..
